# Data Model: Design Gate Runtime Hardening

**Feature**: 141-design-gate-runtime-hardening  
**Date**: 2026-06-02  
**Purpose**: Define the entities, attributes, and validation rules for the design-gate runtime path and the smoke-test bundle.

## No persisted database

This feature has no database. State is file-based: per-iteration Markdown
artifacts, a narrow durable packet file, generated start-prompt text, and the
existing JSON boundary state under `.specrew/`. Entities below are transient
in-process results or on-disk artifacts.

## Entity: Design Analysis Artifact

**Purpose**: The per-iteration `design-analysis.md` decision record, scaffolded
from a template and validated before plan.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| path | string | yes | `specs/<feature>/iterations/<NNN>/design-analysis.md` | On-disk location. |
| problem_framing | section | yes | non-empty, non-placeholder | Problem + constraints. |
| decision_points | section | yes | non-empty | Numbered design decisions. |
| alternatives | section | yes | ≥2 options (Simplest, Reasonable); By-the-book conditional | Option comparison. |
| option_fields | per-option | yes | approach, architectural pattern, quality features considered, effort estimate, reversibility cost, trade-offs, diagram | Required for each option. |
| applicable_lenses | section | optional | read-only references to existing lens files; graceful when none | Dogfoods FR-009. |
| crew_recommendation | section | yes | names exactly one option, non-placeholder | Crew's preferred option. |
| human_decision | section | yes-before-plan | chosen option + reason/modifications + commit hash | Blocks plan until populated. |

### Lifecycle / Relationships

Scaffolded from the template at the design-analysis stop, filled by the Crew,
decided by the human, then consumed as authoritative plan input. Validated by the
Feature 140 validator and by the new callable pre-plan validator.

## Entity: Design-Analysis Gate Packet

**Purpose**: The human approval object for the design-analysis boundary, rendered
from typed fields and (per Option B) persisted as a narrow durable packet.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| typed_fields | object | yes | required human re-entry sections present | Structured input the renderer consumes. |
| rendered_body | markdown | yes | six human re-entry sections; `file:///` references | The human-visible packet. |
| verdict_shape | string | yes | matches `approved for plan with Option <X>` | The expected verdict form. |
| stored_path | string | conditional | under `specs/<feature>/gates/`; design-analysis gate only | Durable 155-lite packet location. |
| from_boundary / to_boundary | string | yes | design-analysis → plan | Boundary metadata. |

### Lifecycle / Relationships

Rendered when the design-analysis stop is presented; validated before the boundary
advances; the chosen option propagates into the Design Analysis Artifact's Human
Decision and into plan input. Scoped to the design-analysis gate only (not the
full Proposal 155 system).

## Entity: Pre-Plan Validation Result

**Purpose**: The transient result returned by the callable pre-plan validator that
gates plan.md authoring.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| valid | bool | yes | true only when artifact + decision valid | Gate outcome. |
| errors | string[] | yes | actionable, name the missing section/field | Why blocked. |
| selected_option | string | when valid | single normalized option token | Chosen option. |

### Lifecycle / Relationships

Computed on demand before plan.md is authored; consumed by coordinator-prompt
enforcement (FR-021). Not persisted.

## Entity: Start/Handoff Packet Path (smoke bundle)

**Purpose**: A generated artifact path inside a start/handoff packet.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| feature_segment | string | yes | non-empty; never `specs//` | Feature directory segment. |
| rendered_url | string | yes | `file:///` form | Clickable reference. |

### Lifecycle / Relationships

Generated by the start-prompt generator; FR-011 requires a non-empty feature
segment or an explicit omission/placeholder.

## Entity: Greenfield Baseline Commit (smoke bundle)

**Purpose**: The baseline commit reference recorded for a fresh greenfield project.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| baseline_hash | string | yes | resolves to a real commit | Provenance anchor. |
| recorded_in | enum | yes | start-context + boundary state consistent | Where it is stored. |

### Lifecycle / Relationships

Established at the first boundary of a fresh greenfield project; FR-013 requires it
to resolve to a real commit and stay consistent across start context and boundary
state.
