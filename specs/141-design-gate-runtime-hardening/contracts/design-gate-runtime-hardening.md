## As-built function surface (Iteration 1)

Implemented by extending `scripts/internal/design-analysis-gate.ps1` (Feature 140 helper; extended, not rewritten):

- `New-SpecrewDesignAnalysisArtifact` / `Get-SpecrewDesignAnalysisTemplatePath` — scaffold the per-iteration artifact from `extensions/specrew-speckit/templates/design-analysis.template.md` (FR-001; non-destructive).
- `Invoke-SpecrewDesignAnalysisPrePlanGate` — callable pre-plan validator; throws `[design-analysis-pre-plan-gate]` when the active substantive artifact, Human Decision, **or durable `gates/` packet** is missing/invalid (FR-002/FR-003/FR-020/FR-021). Smoke-amended 2026-06-02: the packet is now required-in-flow (not an unused helper). Coordinator-prompt enforcement is generated by `scripts/specrew-start.ps1` as an explicit record → render → validate → persist → call sequence; the at-sync plan-boundary gate remains the artifact/decision backstop.
- Decision-commit integrity (FR-008 refinement): the validator rejects a Human Decision whose `Decision recorded in commit` equals the `Design-analysis draft commit`. Template Human Decision uses `Decision verdict` / `Chosen option` / `Design-analysis draft commit` / `Decision recorded in commit`.
- Handoff depth (FR-004 refinement): the template presents a per-option "Design principle / why this matters" rationale.
- `New-SpecrewDesignAnalysisGatePacket` / `Test-SpecrewDesignAnalysisGatePacket` / `Save-SpecrewDesignAnalysisGatePacket` / `Get-SpecrewDesignAnalysisGatePacketPath` — render, validate, and persist the design-analysis gate packet under `specs/<feature>/gates/` only (FR-004/FR-005/FR-006/FR-020).
- `Get-SpecrewDesignAnalysisSelectedOption` — exposes the single human-selected option for plan input (FR-007).
- Validator robustness (FR-022/FR-023): tolerant By-the-book detection in `Get-SpecrewDesignAnalysisOptionBlock` and single-recommendation resolution via `Get-SpecrewDesignAnalysisMarkedOption`.
