# Data Model: Stability and Quality Bundle

**Feature**: 183-stability-quality-bundle
**Date**: 2026-06-16
**Purpose**: Define the local state, config, and evidence entities touched by
Feature 183.

## Entity: HookEventPayload

**Purpose**: Host-provided hook event input consumed by the Specrew dispatcher.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `event` | string | yes | Must map to a supported/verified event for the host. | Hook event name such as SessionStart or a verified Antigravity equivalent. |
| `host_kind` | string | yes | Must be a known host kind. | Host runtime producing the event. |
| `session_id` | string | no | Trim, sanitize, reject blank/malformed values. | Host session identity when available. |
| `raw_payload` | object/string | no | Treat as untrusted; never assume shape. | Host-specific payload body. |

### Lifecycle / Relationships

Created by host hook execution and consumed by dispatcher/provider scripts. It
feeds `SessionKey` resolution and may trigger `FallbackDirective` output.

## Entity: SessionKey

**Purpose**: Filesystem-safe key used for refocus state, journal, dedupe, and
breaker behavior.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `value` | string | yes | Filesystem-safe; must not be global `unknown`. | Resolved host ID or generated per-launch fallback token. |
| `source` | enum | yes | `host-provided` or `per-launch-fallback`. | Explains why the key was chosen. |
| `created_at` | datetime | yes | UTC timestamp. | Diagnostic timestamp. |

### Lifecycle / Relationships

Resolved during hook execution. Existing historical `unknown` files are not
migrated; new writes must avoid global `unknown`.

## Entity: HookFragment

**Purpose**: Provider-generated text fragment included in host hook output.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `kind` | string | yes | Known fragment type. | Bootstrap, refocus, fallback, or diagnostic text. |
| `priority` | integer | yes | Bootstrap priority higher than refocus. | Determines cap-drop order. |
| `text` | string | yes | Non-empty, under host cap after composition. | Output delivered to the host/model. |

### Lifecycle / Relationships

Generated by providers and composed by dispatcher policy. Over-cap composition
drops or shrinks lower-priority fragments first.

## Entity: FallbackDirective

**Purpose**: Minimal governed output emitted when normal provider execution
fails or hook behavior is degraded.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `host_kind` | string | yes | Known host kind. | Host receiving the directive. |
| `text` | string | yes | Non-empty, under cap, no raw stack trace. | User/agent-facing degraded-governed text. |
| `recovery_commands` | string[] | yes | Include relevant commands. | `specrew where`, `/specrew-refocus`, `specrew hooks status`, `specrew start --host <host>`. |

### Lifecycle / Relationships

Created on provider/config/degraded-hook failure. It is a runtime output entity,
not persisted as a new durable schema unless existing journal paths record it.

## Entity: CloseoutDirtySurface

**Purpose**: Classification of repo surfaces that feature-closeout sync uses to
decide commit/push/dashboard messaging.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `paths` | string[] | yes | Repo-relative paths. | Dirty `.specify` extension/config/state surfaces. |
| `classification` | string | yes | Coherent for companion surfaces. | Commit/push/readiness category. |
| `has_upstream` | bool | yes | Derived from git state. | Controls whether "must be pushed" is valid. |
| `dashboard_refreshed` | bool | yes | Must be true on auto-detect paths needing dashboard output. | Dashboard truth evidence. |

### Lifecycle / Relationships

Computed during closeout sync tests and runtime closeout. It does not introduce a
new durable file format; it corrects existing classification behavior.

## Entity: AntigravityHookConfig

**Purpose**: Project-scoped hook configuration managed in `.agents/hooks.json`
for verified Specrew-owned Antigravity hooks.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `path` | string | yes | Must be project-scoped `.agents/hooks.json`. | Config location for this feature. |
| `specrew_entries` | object[] | yes | Only Specrew-owned entries may be added/refreshed/removed. | Hook entries invoking Specrew dispatcher/provider behavior. |
| `user_entries` | object[] | no | Must be preserved. | Existing user hooks not owned by Specrew. |
| `verified_events` | string[] | yes | Only events whose schema/output semantics were verified. | Event bindings safe to claim. |
| `unsupported_events` | string[] | no | Must be labeled degraded/deferred. | Events not claimed as parity. |

### Lifecycle / Relationships

Created or updated by hook deployment/install/update flows. Unsafe parse/merge
must fail open and preserve the existing file. Opt-out/remove behavior follows
the existing hook repair surface.

## Entity: MirrorParityRecord

**Purpose**: Evidence that touched source extension files and deployed
`.specify` mirror files are byte-aligned.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `source_path` | string | yes | Existing source path. | Authoritative file under source extension/runtime tree. |
| `mirror_path` | string | yes | Matching deployed mirror path. | Deployed project copy. |
| `status` | enum | yes | `aligned`, `not-applicable`, or `mismatch`. | Review evidence. |

### Lifecycle / Relationships

Recorded during implementation/review for touched files. Mismatches block
readiness until fixed or explicitly deferred with approval.
