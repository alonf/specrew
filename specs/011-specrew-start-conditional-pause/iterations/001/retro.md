## What Didn't Go Well

- **Session-restart trigger over-inclusivity during planning (2026-05-11)**: Planning incorrectly interpreted the change-detection boundary and included transient handoff files (`.specrew/last-start-prompt.md`, `.specrew/start-context.json`) as files whose changes require a session restart. This created a false-positive pause that was only cleared when Alon Fliess refined the restart-trigger rule. The root cause: planning conflated Copilot-session-loaded prompt files (`.github/agents/*`, `.github/copilot-instructions.md`, `.squad/agents/*/charter.md`, extension templates) with transient runtime handoff files generated by scripts. **Impact**: One unnecessary session-boundary pause during iteration 001 planning. **Prevention**: Known-traps.md row 17 now documents the boundary; future feature planning must verify restart triggers apply only to prompt files, not transient handoff state.

- **Retro scaffolding failed due to missing iteration plan columns (in-session)**: The iteration 001 plan.md file was initially authored without the canonical `Agent` and `Actual` columns required by the retro-scaffolding script (`scaffold-retro-artifact.ps1`). This caused the scaffold command to fail when generating initial retro.md structure. The plan had complete task rows and estimates (Estimated column), but lacked the review columns, causing a schema mismatch. **Impact**: Retro artifact generation blocked; manual repair required before retro scaffolding could proceed. **Prevention**: Iteration plan.md schema must include all eleven columns (Task, Title, Requirement, Story, Effort, Owner, Owner File Globs, Status, Agent, Actual, Verdict) before retro-scaffolding is attempted. Future iteration planning must verify schema completeness before review and retro phases begin.

- **No implementation drift recorded**: This is listed as "didn't go well" only in the sense that it represents zero learning—no unexpected discoveries, no hidden assumptions, no requirement gaps that would normally surface during review. This actually reflects good planning, but the lack of tension-events means no process learning occurred from this iteration.
