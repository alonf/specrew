# Retrospective: PR Lint Scoping (026)

**What worked:** The `-ChangedOnly` scoping design was sound—isolating PR validation to changed iteration directories plus global state surfaces (``.specrew/``, ``.squad/identity/``, ``.specify/feature.json``) correctly balanced latency reduction with governance integrity. The integration test structure (scoped validation + unscoped regression guard + full-repo fallback) correctly framed acceptance criteria. Changelog entry accurately captured the 15min → <1min speedup motivation tied to F-024 PR #306 empirical evidence. **What went wrong:** A long-running implementation agent and the live code drifted from each other during development, then both drifted from the spec/changelog narrative. The repair loop caught three misalignment incidents (fixture behavior vs. spec claim, changelog timing text vs. spec example, live code parameter name vs. integration test assertion). **Process lesson:** When delegating implementation to autonomous agents, checkpoint the live code against the spec/changelog narrative every ~2 commits to surface divergence early. Silent drift between the canonical spec text and the artifact truth (code + tests) only gets visible at final review, when repair work spills into the signoff boundary. In future chore slices, establish a mid-implementation sync point where the implementer re-reads the spec, confirms the changelog entry matches what the code actually does, and verifies the test fixture aligns with both.
