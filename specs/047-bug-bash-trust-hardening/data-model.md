# Data Model: F-047 Trust-Hardening Bug-Bash Bundle

**Feature**: `047-bug-bash-trust-hardening`
**Date**: 2026-05-26
**Purpose**: Define the entities, attributes, and validation rules the 7 items operate on. No persisted database; all state is on-disk governance artifacts, config files, and transient session metadata.

## Entity: HandoffBlock

**Purpose**: The `=== SPECREW HANDOFF ===` … `=== END SPECREW HANDOFF ===` fenced block whose presence/absence relative to a boundary commit is the Item 1/Item 2 detection subject.

### Attributes
| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| present | bool | yes | derived | Whether a handoff block precedes the boundary commit window |
| stopped_at | string | when present | canonical boundary name or `lifecycle-end` | The `STOPPED AT:` value |
| compaction_marker | bool | no | derived from session metadata | True when a compaction event is visible (sub-trigger 3c) |

### Lifecycle / Relationships
Evaluated by `Test-SpecrewHandoffBlockPresent` (new helper). A boundary/iteration commit with `present = false` produces a WARN; with `present = false AND compaction_marker = true` it produces the post-compaction WARN (Item 2 regression lock).

## Entity: ReviewDiagramArtifact

**Purpose**: The `review-diagrams.md` file (Item 3).

### Attributes
| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| exists | bool | — | — | File present in iteration dir |
| has_mermaid_block | bool | when exists | soft-WARN if false | Contains at least one ` ```mermaid ` fence |

### Lifecycle / Relationships
Scaffolded by `scaffold-reviewer-artifacts.ps1` (now with a Mermaid skeleton); validated by `validate-governance.ps1` (soft-WARN on `exists && !has_mermaid_block`).

## Entity: HandoffProse

**Purpose**: Downstream-user-facing coordinator HANDOFF prose (Item 4) — the handoff/decision sections of `scripts/specrew-start.ps1` + `extensions/specrew-speckit/prompts/coordinator-*.md`. (The brief's `installed-instructions/` name does not exist; this entity is the real surface.)

### Attributes
| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| internal_refs | list | — | MUST be empty after audit (in-scope surface only) | Matches of `\bF-\d{3,}\b` / `\bProposal \d{3,}\b` / `\bFeature \d{3,}\b` within handoff prose |

### Lifecycle / Relationships
Audited+rewritten in this feature (handoff prose only); the validator regex WARNs when `internal_refs` is non-empty **inside a `=== SPECREW HANDOFF ===` region**. Internal artifact trees (proposals/, specs/, docs/, tests/, .squad/) are out of scope and legitimately retain feature/proposal numbers.

## Entity: SkillCatalogState

**Purpose**: The computed per-host skill-root state returned by `Get-SpecrewSkillCatalogState` (Item 5).

### Attributes
| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| HasMissingRoots | bool | yes | content-based (was existence-based) | True if any root dir is absent OR present-but-zero-`SKILL.md` |

### Lifecycle / Relationships
Consumed by `specrew start` auto-repair. New rule: an empty skill root sets `HasMissingRoots = true`, so repair fires and the contradictory per-host WARN does not survive.

## Entity: TaskProgressRecord

**Purpose**: Per-task status in `tasks-progress.yml` (Item 7).

### Attributes
| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| status | enum | yes | `planned\|in-progress\|done\|needs-rework\|deferred\|blocked` | Derived from `tasks.md` `[x]` + `state.md` |

### Lifecycle / Relationships
Regenerated by `specrew-start.ps1`; `tasks.md` is the authoritative source. Divergence between `tasks.md` and `state.md` is surfaced, not silently resolved.
