## What Went Well

- **The adversarial doc-consistency sweep paid off.** A fan-out reader-per-doc pass returned zero residual
  defects after the four maintainer-named findings were fixed — the docs reconciliation held up to scrutiny.
- **Close review caught the wrong pressure point before it shipped.** The interim cap fix cut the reconciliation
  excerpt 300→100, spending the fix against the feature's own resume behavior. The maintainer caught it; the real
  cause was the co-resident refocus growing, not resume. Reverted to the dogfooded floor, recovered from refocus,
  and added a floor-guard so it can't recur.
- **The "what is the 10K cap" question produced a better architecture.** Establishing that 10K is the *host* hook-
  output cap (not a model context window) reframed the durable fix: stop inlining what can be a pointer. The
  in-flight disk scan — 100% machine-generated by us — became Proposal 191's lead pilot (write the digest to a
  file, inline only a tiny anti-free-run anchor; ~700–850 char reclaim).

## What Didn't Go Well

- **The interim cap fix spent against core feature behavior.** The first cut reduced the reconciliation excerpt
  300→100 — paying the cap fix against the feature's own resume floor instead of trimming the thing that grew
  (the co-resident refocus B2 tail). Caught in close review, not by the test suite.
- **The masked verdict-worst breach.** The cap test threw at the 9,300 margin assertion before reaching the
  verdict-worst (3b) check, hiding that the true worst case was ~10,090 — *over* the 10K hard cap. A failing
  earlier assertion masked a worse later one.
- **A docs-only iteration absorbed a mid-iteration main merge** and accrued code/test reconciliation drift — so a
  "documentation only" iteration shipped code/test changes, tracked honestly in the drift-log but blurring the
  iteration's stated shape.
- **The merge brought stricter governance that hand-authored artifacts tripped** — first `review.md` (Overall
  Verdict token + Task Verdicts table + Gap Ledger), then `retro.md`'s canonical-section schema (this file, fixed
  at closeout). Hand-authoring governance artifacts after a governance-touching merge is fragile.

## Improvement Actions

- **Never pay a cap fix against core feature behavior.** Trim the thing that *grew* (the refocus B2 tail), not the
  resume directive. The new `DirectiveDeliveryCap` resume-floor guard (handover ≥ 380, reconciliation ≥ 300)
  encodes this so the regression cannot recur silently.
- **Order assertions so the hard limit can't be masked.** A failing margin assertion hid the over-cap
  verdict-worst; the fix had to clear the hard cap, not just the margin. Check the hard ceiling first (or
  independently), so a softer earlier failure never hides a harder later one.
- **Match the scaffolder schema for governance artifacts after a governance-touching merge** (or run the
  scaffolder) — both `review.md` and `retro.md` tripped the stricter post-F-182 schema when hand-authored.
- **Sequence the durable CAP-1 fix after the beta.** Proposal 191's pilot supersedes this iteration's interim
  refocus trim; ship it as a follow-up rather than re-opening F-174.
