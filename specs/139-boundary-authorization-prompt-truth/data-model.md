# Data Model: Boundary Authorization Prompt Truth + Human Re-entry Packet

**Feature**: 139-boundary-authorization-prompt-truth
**Date**: 2026-06-01
**Purpose**: Define the policy, prompt, state, handoff, and evidence entities used to keep generated lifecycle instructions aligned with human-judgment boundary policy.

## Entity: BoundaryPolicyClass

**Purpose**: Represents the lifecycle policy class resolved from `.specrew/config.yml`.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `boundary` | string | yes | Must be a canonical lifecycle boundary name. | Boundary being classified, such as `clarify` or `plan`. |
| `policy_class` | string | yes | Must preserve configured values; `human-judgment-required` triggers human authorization. | Policy decision for the boundary. |
| `source` | string | yes | Must identify `.specrew/config.yml`. | Source used to resolve the policy. |
| `resolved_at` | datetime | yes | ISO 8601 UTC when emitted to state. | Time the start command resolved the snapshot. |

### Lifecycle / Relationships

Created during `specrew start` from `.specrew/config.yml`, serialized into `boundary_enforcement.policy_classes` in `start-context.json`, and consumed by prompt rendering. It links directly to `BoundaryTransition` authorization rules.

## Entity: BoundaryTransition

**Purpose**: Represents a requested lifecycle move from one boundary to the next.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `from_boundary` | string | yes | Must be canonical or `none` for initial entry. | Current lifecycle boundary. |
| `to_boundary` | string | yes | Must be canonical. | Requested next boundary. |
| `requires_human_judgment` | boolean | yes | True when the relevant policy class requires human judgment. | Whether the coordinator must stop. |
| `authorization_evidence` | object/null | conditional | Required unless autonomous mode or explicit bypass applies. | Recorded human verdict evidence. |

### Lifecycle / Relationships

Produced conceptually when a coordinator attempts to advance. It is authorized by `VerdictEvidence` and described to the human through `HumanReentryPacket`.

## Entity: StartContextBoundaryEnforcement

**Purpose**: Stores auditable boundary enforcement state in generated `start-context.json`.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `enabled` | boolean | yes | Must remain true unless explicitly configured otherwise. | Whether boundary enforcement is active. |
| `last_authorized_boundary` | string/null | yes | Must be canonical when present. | Last authorized lifecycle boundary. |
| `pending_next_boundary` | string/null | yes | Must be canonical when present. | Pending next boundary, if any. |
| `policy_classes` | object | yes | Must include resolved classes from `.specrew/config.yml`. | Snapshot used by generated prompt. |
| `verdict_history` | array | yes | Entries must cite boundary movement and evidence hash when available. | Human authorization history. |

### Lifecycle / Relationships

Generated by `specrew start` and updated by boundary sync. `policy_classes` is a snapshot, not the authority; `.specrew/config.yml` remains authoritative.

## Entity: HumanReentryPacket

**Purpose**: Defines the required boundary-stop packet emitted at human-judgment boundaries.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `what_i_just_did` | markdown section | yes | Must summarize meaningful outcome and committed evidence. | Past outcome for human re-entry. |
| `why_i_stopped` | markdown section | yes | Must name exact boundary and concrete reason. | Explains why human judgment is required. |
| `what_needs_your_review` | markdown section | yes | Must include bare `file:///` review targets and exact sections. | Targeted review guidance. |
| `what_happens_next` | markdown section | yes | Must preview phase, artifacts, code/planning status, harder-to-change decisions, and next stop. | Future consequence preview. |
| `discussion_prompts` | markdown section | yes | Must contain 1-3 prompts; targeted prompts need context and default/consequence when relevant. | Human refinement affordance. |
| `what_i_need_from_you` | markdown section | yes | Must list allowed response shapes and explicit approval rule. | Verdict/action request. |

### Lifecycle / Relationships

Rendered before any human-judgment boundary transition. It references `BoundaryTransition`, review artifacts, and `VerdictEvidence`.

## Entity: VerdictEvidence

**Purpose**: Represents explicit human authorization for a boundary transition.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `from_boundary` | string | yes | Must match requested transition. | Starting boundary. |
| `to_boundary` | string | yes | Must match requested transition. | Authorized next boundary. |
| `verdict_text` | string | yes | Must be explicit approval, not generic discussion. | Human authorization text. |
| `authorizing_human` | string | yes | Must identify human. | Person approving the boundary. |
| `auth_commit_hash` | string/null | conditional | Required after boundary commit when available. | Durable evidence commit. |

### Lifecycle / Relationships

Created when the human explicitly approves a boundary. Used by prompt/state checks and the `Status: Approved` contradiction check.

## Entity: SmokeEvidence

**Purpose**: Records beta3 smoke proof for release promotion.

### Attributes

| Attribute | Type | Required | Validation Rules | Description |
| --- | --- | --- | --- | --- |
| `tested_version` | string | yes | Must name beta version. | Specrew build under smoke. |
| `fresh_project_path` | string | yes | Must be a clean downstream project path. | Smoke project location. |
| `stop_boundary` | string | yes | Must record `clarify -> plan` or equivalent stop. | Observed boundary. |
| `plan_preapproval_state` | string | yes | Must state absent/scaffold/non-substantive. | Proof plan did not run early. |
| `packet_excerpt` | markdown/text | yes | Must include six-section evidence excerpt or summary. | Human re-entry packet proof. |
| `result` | enum | yes | `PASS` or `FAIL`. | Smoke outcome. |

### Lifecycle / Relationships

Created during release smoke and committed under the feature's `smoke/` directory before release promotion.
